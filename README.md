# High-Performance Galois LFSR Library (SSE 4.1)

Реализация линейного регистра сдвига с обратной связью (LFSR) над конечными полями $\mathbb{F}_p$ ($p < 256$), экстремально оптимизированная для архитектуры **x86_64** (Intel Whiskey Lake и новее).

## 🚀 Бенчмарки производительности
Результаты получены на процессоре **Intel(R) Core(TM) i7-8565U (Whiskey Lake)** с использованием аппаратного счетчика тактов (`RDTSC`). Компиляция: `g++ -O3 -msse4.1 -march=native`.


| Конфигурация | Операция | Тактов (Median) | Время (ns) | Скорость (Mop/s) |
| :--- | :--- | :---: | :---: | :---: |
| **LFSR (m=4)** | **Next Step (Scalar)** | **7** | 4.09 | **244.7** |
| **LFSR (m=4)** | **Next Step (SSE)** | **6** | 3.83 | **260.7** |
| **LFSR (m=8)** | **Next Step (SSE)** | **12** | 6.54 | **153.0** |
| **LFSR_paired_2x4** | **Next (2 inputs)** | **15** | 8.14 | **122.8 x 2 ch** |
| **LFSR (m=8)** | **Back Step (SSE)** | **25** | 13.23 | **75.6** |
| **LFSR (m=8)** | **Polynomial Mult** | **210** | 106.9 | **9.3** |

## 🛠 Ключевые особенности
*   **Hybrid Path:** Библиотека автоматически выбирает **Scalar** (скалярный) путь для $m \le 4$ (~6-7 тактов) и **SSE** путь для $m > 4$ (~12 тактов).
*   **Zero-Division Policy:** Модульная арифметика реализована через **Barrett Reduction**. Инструкция `idiv` полностью исключена из критических путей, что дает ускорение в 4-5 раз.
*   **Dual-Channel Support:** Класс `LFSR_paired_2x4` позволяет обрабатывать два независимых 4-ячеечных генератора в одном 128-битном регистре с накладными расходами всего в 2-3 такта.
*   **Constant-Time:** Время выполнения `next()` и `back()` не зависит от значений данных, что защищает от атак по времени.
*   **Reversible:** Поддержка шага назад (`back()`) с векторной оптимизацией.

## 🏁 Примеры использования

### Одиночный LFSR (m=8)
Оптимален для работы с длинными регистрами в одном потоке.
```cpp
#include "lfsr8.hpp"

lfsr8::u16x8 K = {1, 3, 0, 2, 0, 1, 1, 4};
lfsr8::LFSR<251, 8> lfsr(K);
lfsr.set_unit_state();

lfsr.next(input); // Выполняется за ~12 тактов
```

### Сдвоенный LFSR (Paired 2x4)
Идеально для параллельной обработки двух независимых потоков данных.

```cpp
#include <iostream>
#include "lfsr8.hpp"

int main() {
    // Коэффициенты для двух генераторов (0-3 и 4-7)
    lfsr8::u16x8 K = {1, 3, 0, 2,  5, 2, 0, 1}; 
    lfsr8::LFSR_paired_2x4<251> paired(K);
    paired.set_unit_state();

    // Обновляем ОБА генератора за ОДИН вызов (~15 тактов)
    paired.next(10, 20); // Вход 10 для первого, 20 для второго

    auto state = paired.get_state();
    std::cout << "Channel 1 (cell 0): " << state[0] << std::endl;
    std::cout << "Channel 2 (cell 4): " << state[4] << std::endl;

    return 0;
}
```

## ⚡ Рекомендации по оптимизации (Compilation Flags)

Для достижения рекордных показателей производительности (Speed of Light), продемонстрированных в бенчмарках, критически важно использовать правильные флаги компилятора:


| Флаг | Зачем он нужен? | Влияние на результат |
| :--- | :--- | :--- |
| `-O3` | Агрессивная оптимизация, инлайнинг и векторизация циклов. | **Критическое** (базовый уровень) |
| `-msse4.1` | Разблокирует аппаратные инструкции `_mm_mullo_epi16/32`. | **Обязательное** (для SSE пути) |
| `-march=native` | Настраивает конвейер (pipeline) под конкретную микроархитектуру (напр. Whiskey Lake). | **Высокое** (~15-30% прироста) |
| `-DNDEBUG` | Отключает `assert` и проверки в конструкторах. | Убирает накладные расходы при инициализации |
