# High-Performance Galois LFSR Library (SSE 4.1)

Реализация линейного регистра сдвига с обратной связью (LFSR) над конечными полями $\mathbb{F}_p$ ($p < 256$, в некоторых условиях $p < 256^2$), экстремально оптимизированная для архитектуры **x86_64** (Intel Whiskey Lake и новее).

## 🚀 Бенчмарки производительности
Результаты получены на процессоре **Intel(R) Core(TM) i7-8565U (Whiskey Lake)** с использованием аппаратного счетчика тактов (`RDTSC`). Компиляция: `g++ -O3 -msse4.1 -march=native`, использовалась WSL, g++ (Ubuntu 12.3.0-1ubuntu1~22.04.3) 12.3.0.


| Конфигурация | Операция | Тактов (Median) | Время (ns) | Скорость (Mop/s) |
| :--- | :--- | :---: | :---: | :---: |
| **LFSR (m=4)** | **Next Step** | **7** | 3.46 | **288.95** |
| **LFSR (m=4)** | **Back Step** | **16** | 8.38 | **119.31** |
| **LFSR (m=8)** | **Next Step** | **11** | 5.74 | **174.24** |
| **LFSR (m=8)** | **Back Step** | **19** | 9.86 | **101.47** |
| **LFSR_paired_2x4** | **Next (2 inputs)** | **14** | 7.14 | **140.03 x 2 ch** |
| **LFSR_paired_2x4** | **Back (2 inputs)** | **15** | 7.39 | **135.40 x 2 ch** |
| **LFSR (m=4)** | **Polynomial Mult** | **60** | 30.69 | **32.59** |
| **LFSR (m=4)** | **Polynomial Square** | **46** | 23.28 | **42.95** |
| **LFSR (m=8)** | **Polynomial Mult** | **251** | 127.13 | **7.87** |
| **LFSR (m=8)** | **Polynomial Square** | **179** | 92.82 | **10.77** |

Здесь:

* Polynomial Mult - умножение состояния генератора на другое состояние по модулю $g(x)$.
* Polynomial Square - возведение состояния генератора в квадрат по модулю $g(x)$.

## 🛠 Ключевые особенности
*   **Dual-Channel Support:** Класс `LFSR_paired_2x4` позволяет обрабатывать два независимых 4-ячеечных генератора в одном 128-битном регистре с симметричной скоростью вперед/назад ('эквивалент 270...280 Mop/s, если бы использовался один LFSR m=4 генератор). Плата за скорость парных генераторов: допускается брать простой модуль $p < 256$ в отличие от одиночного LFSR генератора m=4, для которого $p < 256^2$. 
*   **Constant-Time:** Время выполнения `next()` и `back()` не зависит от значений данных, что защищает от атак по времени.
*   **Reversible:** Поддержка шага назад (`back()`).

## 🏁 Примеры использования

### Одиночный LFSR (m=8)
Оптимален для работы с длинными регистрами в одном потоке.
```cpp
#include "lfsr8.hpp"

lfsr8::u16x8 K = {1, 3, 0, 2, 0, 1, 1, 4};
lfsr8::LFSR<251, 8> lfsr(K);
lfsr.set_unit_state();

lfsr.next(input); // Выполняется за ~12 тактов
```

### Сдвоенный LFSR (Paired 2x4)
Идеально для параллельной обработки двух независимых потоков данных.

```cpp
#include <iostream>
#include "lfsr8.hpp"

int main() {
    // Коэффициенты для двух генераторов (0-3 и 4-7)
    lfsr8::u16x8 K = {1, 3, 0, 2,  5, 2, 0, 1}; 
    lfsr8::LFSR_paired_2x4<251> paired(K);
    paired.set_unit_state();

    // Обновляем ОБА генератора за ОДИН вызов (~15 тактов)
    paired.next(10, 20); // Вход 10 для первого, 20 для второго

    auto state = paired.get_state();
    std::cout << "Channel 1 (cell 0): " << state[0] << std::endl;
    std::cout << "Channel 2 (cell 4): " << state[4] << std::endl;

    return 0;
}
```

## ⚡ Рекомендации по оптимизации (Compilation Flags)

Для достижения рекордных показателей производительности (Speed of Light), продемонстрированных в бенчмарках, критически важно использовать правильные флаги компилятора:


| Флаг | Зачем он нужен? | Влияние на результат |
| :--- | :--- | :--- |
| `-O3` | Агрессивная оптимизация, инлайнинг и векторизация циклов. | **Критическое** (базовый уровень) |
| `-msse4.1` | Разблокирует аппаратные инструкции `_mm_mullo_epi16/32`. | **Обязательное** (для SSE пути) |
| `-march=native` | Настраивает конвейер (pipeline) под конкретную микроархитектуру (напр. Whiskey Lake). | **Высокое** (~15-30% прироста) |
| `-DNDEBUG` | Отключает `assert` и проверки в конструкторах. | Убирает накладные расходы при инициализации |
